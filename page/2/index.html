    };

vue3 学习笔记

2022年04月08日
vue
留言

****### 区别 或者说是新特性

  1. 双向绑定
  • v2 使用的 Object.definePropety()实现,通过 length or keys 修改监听不到
  • v3 使用的基于 Proxy

  1. Dom 渲染

    - v2 使用 diff,对新旧 dom 进行全量对比
- v3 使用 diff 对新旧 dom 对比,v3 不会对静态 dom(没有绑定任何数据)做标记,只对动态 dom(通过 v-bind or {{}} 绑定有数据的)做相应的标记,在数据发生改变时只会对有标记的 dom 进行处理
- 对事件侦听器做了缓存处理,页面不断刷新不会使侦听器销毁再重建
    <template v-for :key>
	<div v-if/>
</template>

  2. 根节点

  • v3 支持多根节点
<template>
	<div/>
	<div/>
</template>
  • v3 添加 render() 函数 JSX 写法
  1. v3 支持多个 v-model (v2 中只支持一个)
  2. 添加内置组件 Suspense
  3. Tree Shaking
  • v2 中一些没用到的内置 api 都会被打包
  • v3 新增 tree shaking 特性,因为 api 都需要导出使用,则一些用不到的 api 不会被打包,主要清除了多余的代码
    比如:import {watch} from 'vue'
  1. 函数式编程
  • 添加 Setup 钩子,也叫 vue Hook

使用

  1. ref 使用
  • 什么是 ref?
    ref 和 reactive 一样,也是用来实现响应式数据的方法
    由于 reactive 必须传递一个对象,所以在实际开发中如果只是想让某个变量实现响应式的时候回非常麻烦
    所以 Vue3 提供了 ref 方法实现简单值得监听
  • 2.ref 本质
    ref 底层其实还是 reactive,所以当运行时系统会自动根据传入的 ref 转换成 reactive.
  • 3.ref 注意点
    在 vue 中使用 ref 的值不用通过 value 获取
    在 js 中使用 ref 的值必须通过 value 获取
import {ref} from 'vue'
/**
 * @event: ref
 * @Tip:
 */
const ref_value = ref({
  name: 'Boring ghost'
})
const ref_change = () => {
  ref_value.value.name = 'Boring ghost is change'
}

<!-- 按钮 -->
<button @click="ref_change">change ref</button>
<div>{{ref_value.name}}</div>
  • Ref 泛型
import { Ref } from 'vue'
const ref_value:Ref<string>=ref("text") // no problem
  • isRef 判断是否为 ref 对象
import { ref,isRef } from 'vue'
const ref_vlue = ref("Boring ghost")
const its_value= "Boring ghost"
isRef(ref_value)//return is true
isRef(its_value)//return is false
  • shallowRef
  • 比较提高性能
  • 监听到 value, value 内部不具备响应式
import { ref, Ref, isRef, shallowRef } from 'vue'
/* shallowRef */
let shallowRef_value = shallowRef({
  name: 'shallowRef Boring ghost'
})
setTimeout(() => {
  shallowRef_value.value.name = '1221' //并不会更新dom上的渲染
  console.log(shallowRef_value.value.name)
  shallowRef_value.value = {  //会更新到dom
    name: '099999'
  }
}, 1000)

<div>{{shallowRef_value.name}}</div> //'099999'
  • triggerRef 强制响应
import { ref, Ref, isRef, shallowRef, triggerRef } from 'vue'
/* shallowRef : 非响应 */
let shallowRef_value = shallowRef({
  name: 'shallowRef Boring ghost'
})
setTimeout(() => {
  shallowRef_value.value.name = '1221'
  console.log(shallowRef_value.value.name)
}, 1000)

/* triggerRef : 强制响应 */
const ref_triggerRef = () => {
  triggerRef(shallowRef_value) //Tip:一般搭配shallowRef使用,因为不会响应到dom,如果需要响应就使用trggerRef强制响应
}
  • customRef 自定义 ref
/* customRef :自定义ref */
function my_ref<T>(value: T) {
  return customRef((trank, trigger) => {
    return {
      get() {
        trank() //收集依赖,让这个值变成响应式
        return value
      },
      set(newvalue: T) {
        value = newvalue
        console.log(value, newvalue)
        trigger() //修改完触发一下以实现效果
      }
    }
  })
}
let cus_value = my_ref('Boring ghost')
cus_value.value = 'new Boring ghost'

  1. to 的用法
  • toRef 原始对象是响应式/非响应式 则 toRef 后的对象就是 响应式/非响应式
  • and toRefs 带有解构
  • 响应式的数据在更新视图时,自然会更新那些带有标记但是因为数据并非是响应式而尚未更新的视图
import { toRef, ref, reactive } from 'vue'
/* toRef: 原始对象是响应式/非响应式 则 toRef 后的对象就是 响应式/非响应式 */
/* Tip: 更改 toRef 后的数据同时会改变原始数据 */
let noRef_obj = {
  name: 'Boring ghost'
}
let ref_zhang = reactive({
  name: 'Boring ghost! zhang'
})
let value_newObj = toRef(noRef_obj, 'name')
let { name } = toRefs(noRef_obj)//toRefs 可以进行解构
let new_zhang = toRef(ref_zhang, 'name')
function chage_obj() {

class Bus{
  list:{[key:string]:Array<Function>}
  constructor(){
    this.list={}
  }
  on(name:string,arg:Function){
    this.list[name]=this.list[name]||[]
    this.list[name].push(arg)
  }
  emit(name:string,arg?:any){
    if(this.list[name]){
      for(let fn of this.list[name]){
        fn(arg)
      }
    }
  }
}
export default new Bus()

  })
}
