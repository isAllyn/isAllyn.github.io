vite 学习笔记

2022年04月08日
插件库
留言

安装

npm init vite@latest

相比webpack

  • vite 直接type=’module’ 引入就预览,而 webpack 打一次包后再预览
  • vite HMR热更新,只刷新更新部分的内容,非所有内容
  • vite Rollip 插件打包

使用ts

  1. 选择 vue-ts 模板时自带 typescript, 只需要npm i

配置 resolve.alias

path会报2307,根据提示install @types/node 即可
vite.config.ts

import { resolve } from 'path'
export default defineConfig({
	resolve:{
		alias:{
			'@':resolve(__dirname,'src')
		}
	}
})

tsconfig.json add+

"baseUrl": ".",
"paths": {
   "@/*": ["src/*"],
   "@page/*": ["src/page/*"],
   "@img/*": ["src/asstes/img/*"]
}

unplugin-auto-import

自动导入插件
https://github.com/antfu/unplugin-auto-import

  • 配置
    import AutoImport from 'unplugin-auto-import/vite'
export default defineConfig({
  plugins: [AutoImport({
      imports: ['vue'],
      dts: 'src/auto-import.d.ts',
    })]
})

  vue(),

提升打包性能

  • 静态文件夹管理

    build: {
    rollupOptions: {
      output: {
        chunkFileNames: 'js/[name]-[hash].js',
        entryFileNames: 'js/[name]-[hash].js',
        assetFileNames: '[ext]/[name]-[hash].[ext]',
      }
    }
  }

  • 压缩插件

    压缩静态资源较大的文件,
    一些js，css，html，以及静态资源。gzip是用算法压缩文件，节省流量。等文件到达浏览器后，浏览器会解压文件，解压出的源文件与gzip之前的大小是没变化的。但是传输过程的流量消耗和带宽节省了。
    npm i -D vite-plugin-compression

    
import viteCompression from 'vite-plugin-compression'

viteCompression({
    verbose: true,
    disable: false,
    threshold: 10240,
    algorithm: 'gzip',
    ext: '.gz',
  })

  • 生产环境移除 console

    build:{
  minify:"terser",
  terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true
      }
  }
}

css前缀

npm i -D autoprefixer

  "defaults", // 默认 "last 2 versions", 
  "last 2 versions",  // 兼容主流浏览器的最近两个版本 "> 1%", 
  "> 1%", // 兼容主流浏览器的最近两个版本 "> 1%", 
  "iOS 7",  // 使用的浏览器需要在市场上的份额大于 1 "iOS 7",
  "last 3 iOS versions" // 兼容 ios 的最新 3 个版本 
postcss.config.js
module.exports = {
  plugins: {
    //如果产生重突就将autoprefixer对象注释掉，因为脚手架内部也默认配置了一样的
        autoprefixer: {
      overrideBrowserslist: [
        "Android 4.1",
        "iOS 7.1",
        "Chrome > 31",
        "ff > 31",
        "ie >= 8",
        "last 10 versions" // 所有主流浏览器最近10版本用
      ],
      grid: true
    },
}

适屏

npm i -S postcss-pxtorem "postcss-pxtorem": "^6.0.0"

flexible 同样改成24

postcss.config.js

  "postcss-pxtorem": {
      rootValue({ file }) {//根元素的值1920 24份为准是80
        return file.indexOf("vant") !== -1 ? 37.5 : 80
      },
      propList: ["*"],
      mediaQuery: false,
      selectorBlackList: ["fiexdPx"],//忽略转换,例如:.fiexdPx_app_page
      minPixelValue: 6,
      exclude: /node_modules/i
    }

module.exports = {
  plugins: {
    //如果产生重突就将autoprefixer对象注释掉，因为脚手架内部也默认配置了一样的
    autoprefixer: {},
    "postcss-pxtorem": {
      rootValue({ file }) {
        return file.indexOf("vant") !== -1 ? 37.5 : 192
      },
      propList: ["*"], //*表示转换所有  height表示值转换高
      exclude: /node_modules/i,
       unitPrecision: 5, //保留rem小数点多少位
       mediaQuery: false, //媒体查询( @media screen 之类的)中不生效
       minPixelValue: 12, //px小于12的不会被转换
       selectorBlackList: ['.radius'],  //则是一个对css选择器进行过滤的数组，比如你设置为['fs']，那例如fs-xl类名，里面有关px的样式将不被转换，这里也支持正则写法。
    }
  }
}
